PROJECT INDEX (Overview for New Agents)
Purpose
- Planner app with backend (Django) and frontend (React+Vite+TypeScript). Key feature: a collaborative Structure Board page where users create boards made of nodes and edges, with modes (Mouse/Pen/Text) for editing and sketching.

Structure
- backend/ (Django project)
  - ixiflowerv2ray: Django settings/urls/wsgi
  - authentication: custom user model and auth endpoints
  - tickets: tasks, submissions, team features, Telegram integration
  - structure: boards API (CRUD, board data with nodes/edges)
- frontend/ (React+Vite TypeScript)
  - src/pages/structure.tsx: Structure Board page (migrating from React Flow to custom Canvas engine)
  - components/ui: Shadcn UI primitives
  - contexts/AuthContext.tsx: Authentication/token
  - config/backend.ts: API base

Data model (Structure Board)
- Board: id (join code), name
- Nodes: { id, type, position: {x,y}, width?, height?, data, style }
  - data may include: label, heading, color, paths (freehand strokes relative to node)
- Edges: { id, source, target, style?, animated? }

Key flows
- Frontend loads boards list, can create/join/switch boards
- Active board auto-saves nodes/edges to backend via PUT /api/structure/boards/{id}/
- Modes: Mouse (select/move), Pen (draw inside nodes), Text (create/edit text)

Running locally
- Backend: Django + requirements.txt; manage.py runserver (or docker-compose)
- Frontend: vite dev (npm run dev) with proxy to backend

Conventions
- Keep backend API stable unless coordinated; prefer enriching node/edge payloads over breaking changes
- Canvas engine resides in frontend/src/pages/structure.tsx in a component named BoardCanvas
- Use TypeScript; keep logic modular inside BoardCanvas where possible
- All AI agents must report what they are doing and keep AGENT.TXT updated continuously after each request

AGENT LOG - Custom Canvas Structure Board

Owner: Rovo Dev
Start: 2026-01-26

Goal
- Replace React Flow usage on the Structure page with a custom Canvas-based implementation using TypeScript and functions only.
- Preserve the existing UI layout and backend APIs as much as possible.
- Support modes (Mouse, Pen, Text), board CRUD, saving/loading, node creation, node selection, node dragging, and drawing confined to nodes.

Plan (Phased Delivery)
Phase 1 (this commit):
- Remove React Flow from the page rendering area and replace with a custom <BoardCanvas> that renders nodes on a single HTMLCanvasElement.
- Keep the right sidebar, boards list, modes, and properties UI.
- Support node creation via drag-and-drop from sidebar into canvas.
- Support single node selection and dragging.
- Support pen mode: draw only inside the node under the cursor; store paths per node; render them.
- Preserve existing backend JSON format for nodes[] and edges[] to minimize server changes.
- Skip node resizing and edge rendering for now. Hide edge property panel since edges aren’t interactable in Phase 1.

Phase 2 (next):
- Add node resize handles (custom), multi-select (shift or box-select), and panning/zooming.
- Add edges (connecting, selecting, styling) on the canvas.
- Add keyboard shortcuts (delete, copy/paste, undo/redo).

Important Decisions
- Rendering: HTML Canvas (better performance for freehand drawing and many elements).
- Data model: keep nodes[]/edges[] shape largely the same (id, position {x,y}, type, data, style). This keeps API compatible.
- Drag-and-drop: reusing existing sidebar drag setup (MIME: 'application/reactflow', label in 'application/reactflow-label').

What’s Done Now
- Introduced a custom BoardCanvas inside structure.tsx that:
  - draws nodes as rounded rectangles filled with node color and heading/label text.
  - clips freehand drawing to the node’s rectangle; freehand paths are stored in node.data.paths.
  - supports selecting a node by clicking; dragging a node by mouse drag in Mouse mode.
  - supports drawing inside a node when Pen mode is active; ignores drawing outside nodes.
  - supports dropping new nodes into canvas.
- Kept the Modes section with Mouse/Pen/Text buttons. Cursor switches to a pen cursor while in Pen mode over canvas.
- Save/Load APIs preserved; nodes/edges arrays persisted as before.
- Edge UI kept but effectively unused in Phase 1.

Next Steps (Phase 2 TODO)
- Implement resize handles per node.
- Implement edges: connection points, line drawing, selection, styling.
- Implement multi-select and marquee selection.
- Implement pan/zoom and fit-to-view.
- Add keyboard shortcuts and a better focus/selection model.

Handover Notes
- All work is currently in frontend/src/pages/structure.tsx.
- Remove any remaining dead React Flow imports after Phase 2 when edges/selection are completed.
- If server validation expects React Flow specific fields, review serializers to ensure saved shape still passes.
